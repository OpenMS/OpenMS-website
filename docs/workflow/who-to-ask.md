# Who to ask

## For website content (news, text, config)

1. Open a **pull request** and request review from someone on the **OpenMS web team** or a repo maintainer with merge rights on `OpenMS-website`.
2. Use GitHub **@mentions** in the PR description if you know who maintains the site.
3. Broader community questions: [OpenMS Discord](https://discord.com/invite/v9tv5BxPch) (link also in site footer).

## For OpenMS software (not the website)

- Installation, workflows, C++ / pyOpenMS: [openms.readthedocs.io](https://openms.readthedocs.io/)
- Code contributions: [OpenMS CONTRIBUTING.md](https://github.com/OpenMS/OpenMS/blob/develop/CONTRIBUTING.md)
- Website contribution overview: [content/en/contribute.md](../../content/en/contribute.md) on the site

## When you need the web team (layout / design / build)

Ask for help if you need to:

- Change colors, fonts, or spacing (`assets/css/`)
- Reorder homepage sections (`layouts/index.html`)
- Edit university partners marquee or sponsors shortcode HTML
- Fix a failing Netlify build you do not understand
- Add a new shortcode or page template

## Repository

- **Website repo:** [github.com/OpenMS/OpenMS-website](https://github.com/OpenMS/OpenMS-website)
- **Issues:** Use GitHub Issues for bugs (broken layout, wrong deploy, broken link on production)
